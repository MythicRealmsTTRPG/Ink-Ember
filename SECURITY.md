# Security Policy

## Supported Versions

Ink & Ember is currently in **pre-1.0 development**.

Because the platform is under active development, **only the latest release is supported**.

| Version | Supported |
| ------- | --------- |
| Latest (pre-1.0) | ✅ |
| Older pre-1.0 releases | ❌ |

We strongly recommend always running the latest version.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly and privately.

### How to Report

- **Do not open a public GitHub issue.**
- Email: `security@yourdomain.com` *(replace with your real address)*  
  **or**
- Use GitHub’s **Private Security Advisory** feature (if enabled).

Please include:

- A clear description of the issue
- Steps to reproduce (or a minimal proof-of-concept)
- The affected version/commit
- Impact assessment (what an attacker could do)
- Any relevant logs, screenshots, or environment details

---

## What to Expect

Because Ink & Ember is early-stage software:

- **Acknowledgement:** within 3 business days
- **Initial assessment:** as soon as reasonably possible
- **Fix timeline:** prioritized by severity and complexity

If accepted, we’ll patch the issue in the next release and may credit you (unless you request anonymity).  
If declined, we’ll explain why it’s not considered a security issue.

---

## Responsible Disclosure

Please avoid:

- Public disclosure before a fix is available
- Accessing data that does not belong to you
- Destructive testing against live environments

If you’re unsure whether something qualifies as a vulnerability, report it anyway.
